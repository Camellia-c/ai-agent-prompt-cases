# Case 03：用评测 Prompt 评估模型回答质量

## 原始需求

我想判断一个模型回答好不好。

用户问题：

AI Agent 为什么需要工具调用？

模型回答：

因为 Agent 很聪明，所以它可以直接完成所有任务。工具调用就是让它变得更智能。

## 问题分析

如果只是让模型“评价一下这个回答”，输出可能会比较主观，也不一定能指出具体问题。

这个原始需求缺少：

1. 明确的评测维度
2. 评分标准
3. 问题定位方式
4. 修改建议
5. 更好的回答版本

所以这里需要设计一个评测 Prompt，让模型按照固定维度评估回答质量，包括相关性、准确性、完整性、清晰度和安全性。

## 优化后的 Prompt

你是一个 AI 输出质量评测员。

你的任务是：评估下面这个模型回答的质量。

请从以下维度评估：

1. 相关性：是否回答了用户问题
2. 准确性：是否存在事实错误
3. 完整性：是否有明显遗漏
4. 清晰度：表达是否容易理解
5. 安全性：是否存在误导、编造或不确定内容

请按照以下格式输出：

## 总体评分

1-5 分，并说明原因。

## 分项评分表

| 维度 | 分数 | 问题说明 |
|---|---:|---|

## 主要问题

列出 2-4 个主要问题。

## 修改建议

说明这个回答应该如何改进。

## 更好的回答版本

给出一版更好的回答。

用户问题：
AI Agent 为什么需要工具调用？

模型回答：
因为 Agent 很聪明，所以它可以直接完成所有任务。工具调用就是让它变得更智能。

## 模型输出

### 总体评分

1 / 5 分

这个回答质量很差。它虽然表面上提到了“工具调用”，但没有真正回答“AI Agent 为什么需要工具调用”。核心观点还存在明显错误：Agent 并不能“直接完成所有任务”，工具调用也不是简单地“让它变得更智能”。

### 分项评分表

| 维度 | 分数 | 问题说明 |
|---|---:|---|
| 相关性 | 1 | 只笼统提到工具调用，没有解释 Agent 为什么需要工具调用。 |
| 准确性 | 1 | “Agent 可以直接完成所有任务”是错误说法。Agent 本身主要依赖模型生成文本，很多任务必须借助外部工具完成。 |
| 完整性 | 1 | 没有提到工具调用的核心原因，如获取实时信息、执行外部操作、调用 API、弥补模型能力边界等。 |
| 清晰度 | 2 | 句子很短，表面上容易读，但概念解释过于空泛，不利于理解。 |
| 安全性 | 1 | 存在误导性表达，可能让用户误以为 Agent 无所不能。 |

### 主要问题

1. 错误夸大 Agent 能力  
“Agent 很聪明，所以它可以直接完成所有任务”是不准确的。Agent 不能凭空查实时数据、操作软件、访问数据库或执行代码。

2. 没有解释工具调用的必要性  
用户问的是“为什么需要工具调用”，回答应该解释工具调用解决了什么问题，而不是只说“让它更智能”。

3. 缺少具体场景  
没有举例说明 Agent 在什么情况下需要调用工具，比如查天气、订票、搜索资料、执行代码、读写文件等。

4. 概念表达太空泛  
“变得更智能”不是准确解释。更准确的说法是：工具调用让 Agent 能连接外部系统，把“想法”转化为“行动”。

### 修改建议

这个回答应该从模型能力边界和 Agent 执行任务需求两个角度展开：

1. 先说明：大模型本身主要负责理解、推理和生成文本。
2. 再说明：很多任务需要外部数据或实际操作，单靠模型做不到。
3. 然后解释：工具调用可以让 Agent 使用搜索、数据库、API、代码执行器、日历、邮件等工具。
4. 最后用例子帮助理解。

### 更好的回答版本

AI Agent 需要工具调用，是因为大模型本身并不能直接完成所有现实任务。

大模型擅长的是理解问题、分析任务、生成回答和制定计划，但它有明显边界：它不能天然获取实时信息，不能直接操作外部软件，不能访问数据库，也不能真正执行代码或完成下单、发邮件、查日历等动作。

工具调用的作用，就是让 Agent 能够连接外部工具和系统，把“会思考”变成“能执行”。

比如：

- 用户问今天北京天气，Agent 需要调用天气 API；
- 用户让它查最新新闻，Agent 需要调用搜索工具；
- 用户让它分析 Excel 数据，Agent 需要读取文件并运行代码；
- 用户让它安排会议，Agent 需要调用日历工具；
- 用户让它查询订单状态，Agent 需要访问业务系统接口。

所以，工具调用不是简单地让 Agent “更聪明”，而是让 Agent 具备获取信息和执行动作的能力。

## 复盘

这个案例的重点是练习“模型回答评测型 Prompt”。

原始需求只是想判断一个回答好不好，但这个需求太宽泛。如果没有明确评测维度，模型可能只会给出笼统评价，例如“这个回答不够完整”。这样的评价不利于定位问题，也不利于后续优化。

这个 Prompt 的优点：

1. 明确了评测角色：AI 输出质量评测员。
2. 给出了五个评测维度：相关性、准确性、完整性、清晰度、安全性。
3. 要求输出总体评分和分项评分表，方便结构化判断。
4. 要求列出主要问题，能定位回答缺陷。
5. 要求给出修改建议和更好的回答版本，形成完整闭环。

从模型输出看，这个 Prompt 能够识别原回答中的核心问题：夸大 Agent 能力、没有解释工具调用必要性、缺少场景例子、概念表达空泛。

可以改进的地方：

1. 可以进一步补充每个评分等级的标准，比如 1 分、3 分、5 分分别代表什么。
2. 可以要求区分“严重问题”和“一般问题”。
3. 可以加入“是否存在幻觉”作为单独维度。
4. 可以要求模型给出“是否通过评测”的最终结论。
