# Case 05：Prompt 对比评测案例

## 原始需求

我想比较两个 Prompt 哪个更好，并说明原因。

对比对象：

### Prompt A

帮我解释一下 RAG。

### Prompt B

你是一个 AI 学习助手。

你的任务是：面向正在准备 AI 评测岗位的学习者，解释 RAG 是什么。

请按照以下结构输出：

1. 一句话定义
2. RAG 解决了什么问题
3. RAG 的基本流程
4. RAG 和普通大模型问答的区别
5. 一个真实业务例子
6. 面试中可能怎么问
7. 1 分钟口头表达版本

要求：

- 用小白能听懂的话解释
- 专业词需要解释
- 不要堆术语
- 输出适合整理进学习笔记
- 如果涉及资料中没有的信息，请说明不确定，不要编造

## 问题分析

如果只问“哪个 Prompt 更好”，模型可能会凭感觉判断，输出比较主观。

所以这里需要使用 Prompt 评分标准，从多个维度进行对比，例如：

1. 目标清晰度
2. 角色设定
3. 背景完整性
4. 输入明确性
5. 执行步骤
6. 输出格式
7. 质量标准
8. 边界控制
9. 可复用性
10. 可评估性

这个案例的重点是：不只是说 Prompt B 更好，而是要说明它为什么更好，以及 Prompt A 应该如何优化。

## 使用的评分方式

本案例使用：

- 大模型评分
- 人工复核

原因：

Prompt 对比属于开放式质量判断，需要理解任务目标、输出结构、适用场景和可复用性。可以先用大模型按照评分标准进行初步评估，再由人工检查评分是否合理。

## 模型输出

总体结论
Prompt B 更好。
核心原因是：Prompt B 明确说明了面向谁、要解释什么、按什么结构输出、用什么语言风格、达到什么质量标准、哪些内容不能乱编。它更贴合用户需求：不仅是“解释 RAG”，而是要输出一份适合 AI 评测岗位学习者整理笔记、准备面试表达的内容。
相比之下，Prompt A 只有一句“帮我解释一下 RAG”，目标太泛，模型可能输出一段普通科普，无法稳定满足“学习笔记 + 面试表达”的需求。

Prompt A 的主要问题


目标太宽泛
“解释一下 RAG”只能说明主题，不能说明解释深度、使用场景和输出目的。


没有说明用户身份
用户是准备 AI 评测岗位的学习者，但 Prompt A 没有体现这一点，模型可能不会结合 AI 评测场景解释。


没有输出结构
模型可能只输出一段定义，缺少流程、业务例子、面试问法、口头表达版本等内容。


没有质量标准
没有要求“用小白能听懂的话”“专业词要解释”“不要堆术语”，所以回答可能过于抽象。


没有边界控制
没有提醒模型“不确定不要编造”，容易出现看似专业但不准确的内容。



Prompt B 的优势


强绑定用户真实需求
它不是单纯解释 RAG，而是面向“AI 评测岗位学习者”，目标更精准。


结构完整，便于整理笔记
从定义、问题、流程、区别、业务例子到面试表达，覆盖了学习和面试两个场景。


降低理解门槛
明确要求用小白能听懂的话解释，并解释专业词，适合初学者。


输出结果更稳定
模型会按照固定结构生成内容，不容易跑偏。


具备评测标准
可以检查回答是否包含 7 个模块、是否避免堆术语、是否适合面试表达。



Prompt A 的优化方向
Prompt A 可以从以下几个方向改进：


补充使用场景
说明不是普通科普，而是用于 AI 评测岗位学习和面试准备。


补充用户背景
说明面向 AI 初学者或准备转岗 AI 评测的人。


规定输出结构
让模型按“定义—问题—流程—区别—案例—面试问法—口头表达”输出。


加入语言要求
要求通俗、具体、少术语，专业词必须解释。


加入边界要求
不确定的地方要说明，不要编造。

## 人工复核

从评分结果看，Prompt B 明显优于 Prompt A。

Prompt A 的问题主要是需求过于模糊，只说明“解释 RAG”，但没有说明面向谁、输出结构、使用场景和质量要求。它可能得到一段普通百科式解释，不一定适合学习笔记或面试表达。

Prompt B 则补充了角色、目标用户、输出结构、质量要求和边界控制，能够让模型更稳定地输出适合 AI 评测岗位学习者使用的内容。

这个案例说明：Prompt 对比不能只凭感觉判断，而应该基于明确的评分维度进行评估。

## 复盘

这个案例的重点是练习“对比型 Prompt”。

通过这个案例可以看到：

1. 低质量 Prompt 通常不是完全不能用，而是输出不可控、不稳定、不适合复用。
2. 高质量 Prompt 会明确角色、任务目标、背景、步骤、输出格式和质量标准。
3. 对比两个 Prompt 时，需要基于评分维度判断，而不是只说“哪个看起来更好”。
4. 对比评测可以使用大模型评分，但最好配合人工复核，避免模型评分过于表面化。
5. Prompt 对比的最终目的不是分胜负，而是找到可优化的方向。

可以改进的地方：

1. 后续可以加入真实模型输出，对比 Prompt A 和 Prompt B 实际生成结果的差异。
2. 可以增加多个 Prompt 版本，例如 A/B/C 三个版本。
3. 可以引入规则评分，比如检查是否包含指定结构、是否包含面试表达、是否说明不确定性。
4. 可以把评分标准进一步量化，例如每个维度 1-5 分对应具体标准。
