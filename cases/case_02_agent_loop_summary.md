# Case 02：用资料总结 Prompt 整理 Agent Loop 学习笔记

## 原始需求

我想学习 Agent Loop，但是资料比较零散，希望能整理成适合学习笔记和面试表达的内容。

## 问题分析

如果只是说“帮我总结 Agent Loop”，模型可能会出现几个问题：

1. 只给百科式解释，不适合面试表达。
2. 只总结概念，不拆解执行流程。
3. 不一定会关联 AI Agent 和 AI 评测岗位。
4. 不一定会指出资料中还缺少什么。
5. 输出结构不稳定，不方便后续整理成学习笔记。

所以这个案例需要设计一个资料总结 Prompt，让模型按照固定结构输出：

- 核心结论
- 关键概念解释
- 重要流程或机制
- 和 AI Agent / AI 评测的关系
- 可能的面试问法
- 1 分钟口头表达版本
- 资料信息不足处

## 优化后的 Prompt

你是一个 AI 学习资料整理助手。

你的任务是：把下面这段资料整理成适合 AI 评测岗位学习者理解、复习和面试表达的笔记。

请按照以下结构输出：

1. 核心结论
2. 关键概念解释
3. 重要流程或机制
4. 和 AI Agent / AI 评测的关系
5. 可能的面试问法
6. 1 分钟口头表达版本

要求：

- 不要照抄原文
- 不要堆术语
- 专业词需要解释
- 重点突出“我面试时怎么说”
- 如果资料中信息不足，请明确说明，不要自行编造
- 输出内容要适合整理进学习笔记

资料如下：

Agent Loop 指的是 AI Agent 在执行任务时反复进行的一套循环流程。一个典型的 Agent Loop 包括理解任务、制定计划、调用工具、观察结果、更新状态、继续执行或结束任务。

普通大模型通常是一次性回答用户问题，而 Agent 更像是一个可以持续工作的任务执行系统。它不会只生成一段回答，而是会根据目标拆解步骤，并在每一步根据当前结果决定下一步做什么。

例如，当用户要求 Agent 帮忙整理一份市场分析报告时，Agent 可能会先理解任务目标，然后规划需要收集哪些信息，再调用搜索工具查找资料，读取网页内容，提取关键信息，生成初稿，最后检查报告是否完整。如果发现信息不足，它可能会继续搜索或向用户提问。

Agent Loop 的关键价值在于让模型从“单轮问答”变成“多步骤任务执行”。在这个过程中，工具调用、记忆、规划和结果检查都很重要。工具调用让 Agent 能获取外部信息或执行操作；记忆让 Agent 能保留上下文；规划让 Agent 知道先做什么后做什么；结果检查则帮助 Agent 判断任务是否完成。

但 Agent Loop 也可能带来问题，例如步骤过多导致成本增加，工具调用错误导致结果偏差，模型判断任务完成的标准不清晰，以及循环次数过多造成效率低下。因此，在实际应用中，需要设计好终止条件、工具使用规则和质量评估标准。

## 模型输出

Agent Loop 学习笔记
1. 核心结论
Agent Loop 是 AI Agent 执行任务时的一套循环机制。
它的核心不是“模型回答一次就结束”，而是让模型围绕一个目标，反复进行：

理解任务 → 制定计划 → 执行动作 → 观察结果 → 更新状态 → 判断是否继续

所以，Agent Loop 的价值在于：
让大模型从“单轮问答工具”，变成一个可以持续推进任务的执行系统。
面试时可以这样说：

普通大模型更像是回答问题，而 Agent 更像是完成任务。Agent Loop 就是支撑 Agent 持续工作的循环流程，它让模型可以根据目标拆解任务、调用工具、观察反馈，并不断决定下一步行动，直到任务完成或满足终止条件。


2. 关键概念解释
2.1 Agent Loop 是什么？
Agent Loop 可以理解为 Agent 的工作循环。
它不是一次性生成答案，而是让模型不断判断：


当前任务是什么？


下一步应该做什么？


是否需要调用工具？


工具返回的结果是否有用？


任务是否已经完成？


是否还需要继续执行？


简单说：

Agent Loop 就是 Agent “边做边看、边看边改、直到完成任务”的过程。


2.2 AI Agent 和普通大模型有什么区别？
普通大模型通常是：

用户问一句 → 模型答一句

而 Agent 更像是：

用户给一个目标 → Agent 自己拆解步骤 → 调用工具 → 检查结果 → 继续推进

比如用户说：“帮我整理一份市场分析报告。”
普通大模型可能直接生成一份报告。
Agent 可能会：


先理解报告主题和目标；


判断需要收集哪些市场信息；


调用搜索工具查资料；


阅读网页内容；


提取重点；


生成报告初稿；


检查有没有信息缺口；


必要时继续搜索或询问用户。


所以区别在于：
对比项普通大模型AI Agent工作方式一次性回答多步骤执行是否会规划通常较弱会拆解任务是否调用工具不一定经常需要是否检查结果不一定需要持续判断适合任务问答、总结、改写搜索、分析、执行、自动化流程

2.3 工具调用是什么？
工具调用指的是 Agent 不只靠模型自身知识回答，而是可以使用外部工具完成任务。
比如：


搜索网页；


读取文件；


调用 API；


查询数据库；


操作表格；


生成报告。


面试表达：

工具调用让 Agent 不只停留在文本生成层面，而是可以获取外部信息或执行具体操作。它是 Agent 能完成复杂任务的重要能力。


2.4 记忆是什么？
这里的记忆不是人类意义上的记忆，而是指 Agent 在任务过程中能够保存和利用上下文信息。
比如：


用户的目标；


已经完成了哪些步骤；


工具返回了哪些结果；


哪些信息还缺失；


当前任务状态是什么。


面试表达：

记忆可以帮助 Agent 保留上下文，避免每一步都从零开始。它让 Agent 能知道自己已经做过什么、接下来还要做什么。


2.5 规划是什么？
规划就是 Agent 根据目标，把大任务拆成小步骤。
比如“写市场分析报告”可以拆成：


明确行业；


收集市场规模；


分析竞品；


总结用户需求；


输出报告结构；


生成最终内容。


面试表达：

规划能力决定了 Agent 是否能有条理地推进复杂任务，而不是想到哪做到哪。


2.6 结果检查是什么？
结果检查就是 Agent 判断当前执行结果是否满足任务要求。
例如：


信息是否完整？


工具返回结果是否可靠？


报告结构是否完整？


是否还需要继续查资料？


是否应该结束任务？


面试表达：

结果检查是 Agent Loop 中很关键的一步，因为 Agent 不能无限循环，也不能在结果不完整时提前结束，所以需要有质量判断和终止条件。


3. 重要流程或机制
一个典型 Agent Loop 可以拆成 6 步：

第一步：理解任务
Agent 先判断用户真正想完成什么。
例如用户说：

帮我整理一份市场分析报告。

Agent 需要理解：


报告主题是什么？


面向谁使用？


需要多详细？


是否需要最新数据？


是否需要引用来源？



第二步：制定计划
Agent 根据目标拆解任务路径。
比如：

先查行业背景，再查市场规模，再分析竞品，最后输出报告。

这一步的作用是避免 Agent 直接乱写。

第三步：调用工具
如果模型自身信息不够，Agent 需要调用工具。
例如：


用搜索工具查最新资料；


用文件读取工具查看文档；


用计算工具处理数据；


用数据库工具查询记录。



第四步：观察结果
工具返回结果后，Agent 需要判断这些结果是否有价值。
比如：


搜索结果是否相关？


数据是否过时？


内容是否和任务目标匹配？


有没有明显缺失？



第五步：更新状态
Agent 根据新结果更新当前任务进度。
例如：

已经拿到市场规模数据，但缺少竞品信息，所以下一步继续查竞品。

这一步让 Agent 不会忘记前面做过什么。

第六步：继续执行或结束任务
Agent 最后要判断：


任务是否已经完成？


结果是否达到要求？


是否还需要继续调用工具？


是否应该向用户提问？


是否应该输出最终答案？


如果还没完成，就进入下一轮循环。
如果完成了，就结束任务并输出结果。

4. 和 AI Agent / AI 评测的关系
4.1 和 AI Agent 的关系
Agent Loop 是 AI Agent 的核心运行方式之一。
可以这样理解：

Agent 之所以像“智能体”，不是因为它只会聊天，而是因为它能围绕目标持续行动。Agent Loop 就是它持续行动的基本框架。

Agent Loop 把以下能力串起来：


理解任务：知道用户要什么；


规划：知道先做什么、后做什么；


工具调用：能获取信息或执行操作；


记忆：能保留过程状态；


结果检查：能判断是否完成；


终止条件：知道什么时候停下来。


所以，Agent Loop 是 Agent 从“能说”变成“能做”的关键机制。

4.2 和 AI 评测的关系
AI 评测岗位不只是看最终答案对不对，还要看 Agent 在执行过程中的表现。
对于 Agent Loop，评测时可以关注这些点：
评测维度关注问题任务理解Agent 是否正确理解用户目标？计划合理性步骤拆解是否清晰、有效？工具调用是否调用了合适的工具？调用参数是否正确？结果观察是否正确理解工具返回结果？状态更新是否记住已经完成的步骤？纠错能力工具失败或信息不足时，是否能调整策略？终止判断是否在合适的时候结束？有没有过早结束或无限循环？成本效率是否调用了过多工具，导致成本和时间浪费？输出质量最终结果是否完整、准确、符合用户需求？
面试时可以这样说：

在 AI 评测里，Agent Loop 的评测不能只看最终答案，还要看中间过程。比如它是否正确理解任务、是否制定合理计划、是否调用了正确工具、是否能根据工具结果调整下一步，以及是否有明确的终止条件。因为 Agent 的问题往往不是单点错误，而是流程中的某一步出错导致最终结果偏差。


5. 可能的面试问法
问法 1：什么是 Agent Loop？
可以回答：

Agent Loop 是 AI Agent 执行任务时的循环流程。它通常包括理解任务、制定计划、调用工具、观察结果、更新状态、判断是否继续执行或结束。它的作用是让模型不只是一次性回答问题，而是能够围绕目标进行多步骤任务执行。


问法 2：Agent 和普通大模型有什么区别？
可以回答：

普通大模型通常是用户问一句，模型回答一句；而 Agent 更强调任务执行。Agent 会先理解目标，再拆解步骤，必要时调用工具，根据工具结果调整行动，最后判断任务是否完成。所以 Agent 更适合处理搜索、分析、自动化操作这类复杂任务。


问法 3：Agent Loop 里最重要的环节有哪些？
可以回答：

我认为比较关键的是规划、工具调用、结果观察和终止判断。规划决定任务能不能被合理拆解；工具调用决定 Agent 能不能获取外部信息或执行动作；结果观察决定它能不能理解反馈；终止判断决定它是否能在合适的时候停下来，避免无限循环或提前结束。


问法 4：Agent Loop 可能出现哪些问题？
可以回答：

常见问题包括：任务理解错误、计划不合理、工具调用错误、工具返回结果理解错误、上下文记忆丢失、循环次数过多、成本过高，以及任务完成标准不清晰。这些问题都可能导致 Agent 最终输出不准确，或者执行效率很低。


问法 5：如果你来评测一个 Agent Loop，你会看哪些指标？
可以回答：

我会从过程和结果两个角度看。过程上看它是否正确理解任务、计划是否合理、工具调用是否正确、是否能根据结果调整策略、是否有清晰的终止条件。结果上看最终答案是否准确、完整、符合用户需求。同时也会关注成本和效率，比如是否有无意义的重复调用。


问法 6：为什么 Agent Loop 需要终止条件？
可以回答：

因为 Agent Loop 是循环机制，如果没有终止条件，Agent 可能会一直调用工具或反复检查，导致效率低、成本高。终止条件可以帮助 Agent 判断什么时候任务已经完成，什么时候需要停止，什么时候应该转向向用户提问。


6. 1 分钟口头表达版本
面试时可以这样说：

Agent Loop 是 AI Agent 执行任务时的一套循环流程。普通大模型通常是一次性回答用户问题，而 Agent 更像是一个任务执行系统。它会先理解用户目标，再制定计划，必要时调用工具获取外部信息或执行操作，然后观察工具返回结果，更新当前任务状态，并判断下一步是继续执行、调整策略，还是结束任务。
这个机制的价值在于，它让大模型从单轮问答变成多步骤任务执行，适合处理搜索、分析、报告生成、自动化操作等复杂任务。
但 Agent Loop 也会带来评测难点，比如工具调用是否正确、计划是否合理、是否能理解工具结果、是否会重复循环、是否有明确的终止条件。所以在 AI 评测里，不能只看最终答案，还要评估 Agent 执行过程中的每一步是否合理。


资料信息不足处
这段资料已经能说明 Agent Loop 的基本概念和价值，但如果要进一步深入面试，还缺少这些内容：
缺少内容为什么重要具体技术实现方式比如 ReAct、Plan-and-Execute 等 Agent 模式工具调用失败时的处理机制面试可能会问 Agent 如何容错评测指标的量化方式比如成功率、工具调用准确率、任务完成率等真实案例日志分析AI 评测岗位可能会看 Agent 执行轨迹并定位问题和 RAG、Function Calling 的区别面试中容易混问，需要后续单独整理
所以这份资料适合作为 Agent Loop 入门笔记，但还不够支撑深度技术面试。下一步可以继续补充：ReAct 实现原理、工具调用评测、Agent 执行轨迹评测方法。


## 复盘

这个案例的重点是练习“资料总结型 Prompt”。

原始需求只是想学习 Agent Loop，但这个需求本身比较模糊。如果直接问模型，输出可能只是概念解释，不一定适合面试，也不一定能关联 AI 评测岗位。

这个 Prompt 的优点：

1. 明确了角色：AI 学习资料整理助手。
2. 明确了目标：整理成适合 AI 评测岗位学习者理解、复习和面试表达的笔记。
3. 固定了输出结构，让结果更稳定。
4. 要求专业词解释，降低理解门槛。
5. 加入“面试问法”和“1 分钟口头表达”，方便后续求职准备。
6. 要求说明资料不足，避免模型在信息不足时编造内容。

从模型输出看，这个 Prompt 能够把 Agent Loop 拆成概念、流程、和 AI Agent 的关系、和 AI 评测的关系、面试问法等模块，比较适合整理进学习笔记。

可以改进的地方：

1. 可以在 Prompt 中明确要求使用 Markdown 表格，避免模型输出的表格格式混乱。
2. 可以补充“请区分资料中已有信息和模型推理补充的信息”。
3. 可以增加“输出不要太长，控制在 1500 字以内”之类的限制。
4. 可以单独要求输出“评测维度表”，更贴近 AI 评测岗位。
