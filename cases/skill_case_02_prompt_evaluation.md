# Skill Case 02：Prompt 质量评测

## 1. 使用的 Skill

`skills/prompt_evaluation_skill.md`

## 2. 案例目标

对比两个 Prompt 的质量，判断哪个更适合完成用户需求，并说明原因。

## 3. 用户需求

我想学习 RAG，并希望输出内容适合整理成 AI 评测岗位的学习笔记和面试表达。

## 4. Prompt A

帮我解释一下 RAG。

## 5. Prompt B

你是一个 AI 学习助手。

你的任务是：面向正在准备 AI 评测岗位的学习者，解释 RAG 是什么。

请按照以下结构输出：

1. 一句话定义；
2. RAG 解决了什么问题；
3. RAG 的基本流程；
4. RAG 和普通大模型问答的区别；
5. 一个真实业务例子；
6. 面试中可能怎么问；
7. 1 分钟口头表达版本。

要求：

- 用小白能听懂的话解释；
- 专业词需要解释；
- 不要堆术语；
- 输出适合整理进学习笔记；
- 如果涉及资料中没有的信息，请说明不确定，不要编造。

## 6. 适用原因

该任务不是单纯让模型生成一个新 Prompt，而是要评估 Prompt 的质量、适配度和可执行性，因此适合使用 Prompt 质量评测 Skill。

## 7. 预期评测维度

根据 Skill 要求，应从以下维度评估：

| 维度 | 说明 |
|---|---|
| 目标清晰度 | 是否明确用户真正想完成什么 |
| 角色设定 | 是否设置了合适的模型角色 |
| 输出结构 | 是否规定了清晰的输出格式 |
| 约束条件 | 是否说明质量要求和限制 |
| 可执行性 | 模型是否容易按照要求执行 |
| 用户需求匹配度 | 是否贴合学习笔记和面试表达目标 |

## 8. 示例结论摘要

Prompt B 更适合完成用户需求。

原因是 Prompt B 明确了目标用户、学习场景、输出结构和质量要求。它不只是让模型解释 RAG，而是要求模型面向 AI 评测岗位学习者，输出可整理、可复习、可面试复述的内容。

Prompt A 虽然简单直接，但缺少用户背景、输出结构和质量约束，结果更容易变成普通概念解释。

## 9. 评测关注点

使用该 Skill 时，需要检查：

- 是否说明哪个 Prompt 更好；
- 是否按维度评分；
- 是否解释评分依据；
- 是否指出 Prompt 的具体问题；
- 是否给出可执行的优化建议；
- 是否生成更好的 Prompt 版本。
