# Case 06：Prompt 优化改进案例

## 原始需求

我想把一个模糊 Prompt 优化成更清晰、可执行、可复用的 Prompt。

## 原始 Prompt

帮我总结这篇文章。

## 问题分析

这个 Prompt 的问题是：任务太模糊，缺少必要约束。

主要缺失包括：

1. 没有说明总结对象  
   不知道是给小白、专业人士、面试者、业务人员，还是管理者看。

2. 没有说明总结目的  
   不知道是为了学习、汇报、复习、写作、决策，还是面试表达。

3. 没有说明输出结构  
   模型可能输出一整段文字，也可能输出几个要点，结果不稳定。

4. 没有说明质量标准  
   不知道是否需要简洁、准确、口语化、专业、避免照抄原文。

5. 没有说明信息不足时如何处理  
   如果文章内容缺少关键信息，模型可能会自行补充甚至编造。

6. 可复用性弱  
   只能针对一次任务使用，不能稳定迁移到其他资料总结场景。

## 使用的优化原则

参考 `notes/prompt_scoring_rubric.md` 中的 Prompt 评分维度，本次优化重点补充：

1. 目标清晰度
2. 角色设定
3. 背景完整性
4. 输入明确性
5. 执行步骤
6. 输出格式
7. 质量标准
8. 边界控制
9. 可复用性
10. 可评估性

## 优化方向

| 维度 | 原 Prompt 问题 | 优化方式 |
|---|---|---|
| 目标清晰度 | 只说“总结”，不知道总结成什么 | 明确整理成学习笔记和面试表达 |
| 角色设定 | 没有角色 | 设置为 AI 学习资料整理助手 |
| 背景完整性 | 不知道面向谁 | 明确面向 AI 评测岗位学习者 |
| 输入明确性 | 没有指定输入位置 | 使用“资料如下：【粘贴资料】” |
| 执行步骤 | 没有步骤 | 要求按核心结论、概念解释、流程、关系、面试问法输出 |
| 输出格式 | 没有格式要求 | 规定分标题输出 |
| 质量标准 | 没有质量要求 | 要求不照抄、不堆术语、专业词解释 |
| 边界控制 | 可能编造 | 要求信息不足时明确说明 |
| 可复用性 | 只能用于当前文章 | 设计成可替换资料的模板 |
| 可评估性 | 不方便判断好坏 | 输出结构固定，方便检查是否完整 |

## 优化后的 Prompt

你是一个 AI 学习资料整理助手。

你的任务是：把下面这段资料整理成适合 AI 评测岗位学习者理解、复习和面试表达的笔记。

请按照以下结构输出：

1. 核心结论
2. 关键概念解释
3. 重要流程或机制
4. 和 AI Agent / AI 评测的关系
5. 可能的面试问法
6. 1 分钟口头表达版本
7. 资料信息不足处

要求：

- 不要照抄原文
- 不要堆术语
- 专业词需要解释
- 重点突出“我面试时怎么说”
- 如果资料中信息不足，请明确说明，不要自行编造
- 输出内容要适合整理进学习笔记

资料如下：

【粘贴资料】

## 优化后 Prompt 的预期效果

优化后的 Prompt 应该能让模型：

1. 更清楚地知道任务目标  
   不是泛泛总结，而是整理成 AI 评测岗位学习笔记。

2. 输出更稳定  
   因为规定了固定结构。

3. 更适合目标用户  
   明确面向 AI 评测岗位学习者，而不是泛泛读者。

4. 更方便复用  
   只需要替换“资料如下”中的内容，就能用于不同文章或资料。

5. 更容易评估  
   可以检查输出是否包含核心结论、概念解释、流程、面试问法等模块。

6. 更不容易编造  
   因为明确要求资料不足时说明缺口。

## 复盘

这个案例的重点是练习“优化型 Prompt”。

原始 Prompt 并不是完全不能用，但它缺少角色、目标、背景、步骤、格式、质量标准和边界控制，所以模型输出不可控，也不方便复用。

优化后的 Prompt 通过补充上下文和结构，让模型更容易稳定完成任务。

这个案例说明：

1. Prompt 优化不是把句子写长，而是补齐任务执行所需的信息。
2. 优化 Prompt 时要先分析原 Prompt 缺什么，再针对性补充。
3. 好 Prompt 应该让模型知道“为谁做、做什么、怎么做、输出成什么、什么不能做”。
4. 对 AI 评测岗位来说，Prompt 优化能力和输出质量评估能力是连在一起的。
5. 后续还可以结合真实模型输出，对比优化前后结果差异。

可以改进的地方：

1. 可以加入真实文章材料，分别用原 Prompt 和优化 Prompt 跑一次，比较实际输出差异。
2. 可以让大模型先评分，再由人工复核评分是否合理。
3. 可以进一步加入字数限制、表格格式、引用要求等约束。
4. 可以扩展为多轮优化：原 Prompt → 优化版 1 → 优化版 2 → 最终版。
