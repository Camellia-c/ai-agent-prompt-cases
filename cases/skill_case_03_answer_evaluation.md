# Skill Case 03：AI 回答质量评测

## 1. 使用的 Skill

`skills/ai_answer_evaluation_skill.md`

## 2. 案例目标

评估一个模型回答是否准确、完整、清晰，并给出修改建议和更好的回答版本。

## 3. 用户问题

AI Agent 为什么需要工具调用？

## 4. 模型回答

因为 Agent 很聪明，所以它可以直接完成所有任务。工具调用就是让它变得更智能。

## 5. 适用原因

该任务需要判断模型回答质量，不是简单改写回答。

需要从相关性、准确性、完整性、清晰度、安全性和可操作性等维度进行评估，因此适合使用 AI 回答质量评测 Skill。

## 6. 预期评测维度

| 维度 | 评估重点 |
|---|---|
| 相关性 | 是否回答了用户关于工具调用原因的问题 |
| 准确性 | 是否准确解释 Agent 和工具调用的关系 |
| 完整性 | 是否说明工具调用解决什么问题 |
| 清晰度 | 表达是否容易理解 |
| 安全性 | 是否存在误导性说法 |
| 可操作性 | 是否能帮助用户形成可复用理解 |

## 7. 示例结论摘要

该回答质量较低。

主要问题是它把 Agent 需要工具调用的原因归结为“Agent 很聪明”，这个说法不准确。工具调用的核心原因不是 Agent 聪明，而是大模型本身存在边界：不能直接访问实时信息、不能读取外部文件、不能操作软件系统，也不擅长高精度计算。

工具调用让 Agent 能够使用搜索、代码执行、文件读取、数据库、邮件日历等外部能力，从而完成更复杂的真实任务。

## 8. 更好的回答方向

更好的回答应包括：

1. 说明大模型本身的能力边界；
2. 解释工具调用的作用；
3. 举具体例子，如搜索、计算、读取文件、操作日历；
4. 关联 Agent Loop，说明工具调用处在执行和观察结果阶段；
5. 避免“Agent 可以完成所有任务”这类绝对化说法。

## 9. 评测关注点

使用该 Skill 时，需要检查：

- 是否指出原回答的核心错误；
- 是否按维度评分；
- 是否区分相关性和准确性；
- 是否指出误导性表达；
- 是否给出具体修改建议；
- 是否提供更好的回答版本。
