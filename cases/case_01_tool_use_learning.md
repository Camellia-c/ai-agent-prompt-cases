# Case 01：用 Prompt 学习 Tool Use / Function Calling

## 原始需求

我想学习 Tool Use / Function Calling。

## 问题分析

如果只问“Tool Use 是什么”，模型可能会回答得太泛，不一定适合面试表达，也不一定能结合 AI Agent 场景。

所以这里需要把需求改造成一个更清晰的 Prompt，让模型按照固定结构输出，包括定义、作用、流程、业务例子和面试表达。

## 优化后的 Prompt

你是一个 AI Agent 学习助手。

你的任务是：帮助一个计算机基础一般、正在准备 AI 评测岗位的人，学习“Tool Use / Function Calling”。

请按照以下结构输出：

1. 一句话定义
2. 它解决了什么问题
3. 它的基本工作流程
4. 在 AI Agent 里的作用
5. 一个真实业务例子
6. 面试中可能怎么问
7. 一段 1 分钟口头回答稿

要求：
- 用小白能懂的话解释
- 专业词要解释
- 不要写得太像百科
- 输出适合整理进学习笔记

## 模型输出

### 1. 一句话定义

Tool Use / Function Calling，指的是大模型在回答问题时，不只依赖自己已有的知识生成文字，而是可以根据任务需要，调用外部工具、API、数据库、搜索引擎、计算器或业务系统，获取信息或执行操作。

简单说就是：

> 模型自己不会真正查天气、订会议、算复杂数据，但它可以调用对应工具来完成这些事。

---

### 2. 它解决了什么问题

普通大模型主要靠训练时学到的知识生成回答，但这会有几个问题：

1. 知识可能过时  
   比如最新价格、新闻、政策、接口文档变化，模型不一定知道。

2. 不能直接操作外部系统  
   比如发邮件、查日历、读文件、调用数据库，这些不是单靠生成文字能完成的。

3. 精确任务容易出错  
   比如复杂计算、数据统计、代码运行，模型直接心算可能不稳定。

4. 很难完成真实业务闭环  
   真实工作不是只回答问题，还需要查资料、分析数据、调用系统、生成结果。

Tool Use / Function Calling 就是为了解决这些问题，让模型从“只会说”变成“能连接工具做事”。

---

### 3. 它的基本工作流程

一个典型的工具调用流程是：

1. 用户提出任务  
   例如：“帮我查一下今天北京天气，并提醒我是否需要带伞。”

2. 模型理解任务  
   模型判断这个问题需要最新天气信息，不能只靠记忆回答。

3. 模型选择工具  
   选择天气查询工具。

4. 模型生成工具调用参数  
   例如：
   - 地点：北京
   - 日期：今天

5. 外部工具返回结果  
   例如：北京今天有小雨，气温 18-24℃。

6. 模型整合工具结果并回答用户  
   例如：“今天北京有小雨，建议带伞，早晚温差不大。”

---

### 4. 在 AI Agent 里的作用

Tool Use 是 AI Agent 能真正执行任务的关键能力之一。

AI Agent 一般可以理解成：

> 能感知任务、规划步骤、调用工具、执行操作、根据结果继续调整的系统。

其中工具调用对应的是“执行能力”。

如果没有 Tool Use，Agent 只能停留在“给建议”的层面。  
有了 Tool Use，Agent 才能完成更接近真实工作的任务，比如：

- 搜索资料
- 读取文件
- 分析表格
- 调用接口
- 运行代码
- 生成报告
- 发送邮件
- 创建日程

所以 Tool Use 是 Agent 从“聊天机器人”走向“任务执行助手”的重要基础。

---

### 5. 一个真实业务例子

假设一个客服质检 Agent 要检查客服聊天记录。

用户任务是：

> 帮我分析昨天所有客服对话，找出服务态度差、没有解决问题、存在违规承诺的案例。

这个 Agent 可能需要这样工作：

1. 调用数据库工具，读取昨天的客服聊天记录。
2. 调用文本分析工具，识别用户投诉、客服回复和问题结果。
3. 按照质检规则判断是否存在问题。
4. 生成评分表和问题摘要。
5. 输出需要人工复查的高风险案例。

这里模型本身不是“凭空知道”聊天记录的，而是通过工具读取数据，再结合评测标准进行分析。

---

### 6. 面试中可能怎么问

面试中可能会这样问：

1. 什么是 Tool Use / Function Calling？
2. 大模型为什么需要工具调用？
3. Tool Use 和普通 Prompt 有什么区别？
4. Function Calling 的基本流程是什么？
5. Tool Use 在 AI Agent 中起什么作用？
6. 工具调用可能会出现哪些问题？
7. 如果工具返回错误结果，Agent 应该怎么办？
8. Tool Use 和 RAG 有什么区别？

---

### 7. 一段 1 分钟口头回答稿

Tool Use，也叫 Function Calling，指的是大模型在完成任务时，可以根据用户需求调用外部工具或接口，比如搜索、数据库、计算器、代码执行器、文件读取工具等。

它解决的核心问题是：大模型本身只擅长理解和生成文本，但它不一定知道最新信息，也不能直接操作外部系统。所以通过工具调用，模型可以先判断任务需要什么工具，再生成对应参数，让外部工具执行，最后把工具返回的结果整合成用户能理解的回答。

在 AI Agent 里，Tool Use 很重要，因为 Agent 不只是聊天，它要完成真实任务。比如一个客服质检 Agent，需要读取聊天记录、分析问题、打分、生成报告，这些都离不开工具调用。可以说，Tool Use 是 Agent 从“能回答”走向“能执行”的关键能力。

## 复盘

这个案例的重点不是单纯解释 Tool Use，而是练习如何把一个模糊学习需求改造成结构化 Prompt。

这个 Prompt 的优点：

1. 明确了角色：AI Agent 学习助手。
2. 明确了学习对象：计算机基础一般、准备 AI 评测岗位的人。
3. 固定了输出结构，避免模型回答散乱。
4. 加入了真实业务例子，方便理解 Agent 场景。
5. 加入了面试问法和口头回答稿，适合后续求职准备。

可以改进的地方：

1. 后续可以加入“Tool Use 和 RAG 的区别”。
2. 可以加入“工具调用失败时如何处理”。
3. 可以进一步补充 Function Calling 的 JSON 参数结构示例。
