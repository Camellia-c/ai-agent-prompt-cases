# Case 07：Agent Workflow Prompt 综合案例

## 原始需求

我想设计一个完整的 Agent Prompt，让 Agent 不只是判断是否需要工具，还能按照完整任务流程执行，包括理解任务、制定计划、判断工具、观察结果、更新状态、判断是否结束并输出结果。

## 测试用户问题

帮我判断：如果我要写一篇关于 AI Agent 工具调用的学习笔记，是否需要搜索资料？应该怎么执行？

## 问题分析

这个任务不是单纯的概念解释，也不是单纯的搜索任务。

它包含几个子问题：

1. 用户想写一篇学习笔记。
2. 主题是 AI Agent 工具调用。
3. 用户想判断是否需要搜索资料。
4. 用户还想知道应该怎么执行。

因此，Agent 需要完成：

- 理解用户目标
- 判断任务类型
- 判断是否需要工具
- 说明搜索资料是否必要
- 给出执行计划
- 说明如果不搜索可以写到什么程度
- 说明如果搜索可以补充什么信息
- 给出最终执行方案

第一步：理解笔记目标

这篇学习笔记不应该只回答“工具调用是什么”，而应该解决三个问题：

它是什么？
Agent 为什么需要它？
AI 评测岗位怎么评价它做得好不好？

所以笔记不能只写概念，还要写流程、场景、风险和评测维度。

第二步：判断是否需要搜索

基础内容可以直接写，例如：

工具调用就是让大模型在只生成文本之外，可以调用搜索、计算器、数据库、API、代码执行器等外部工具，从而完成自己原本不能直接完成的任务。

但这只是入门理解。

如果要写成更专业的学习笔记，就需要搜索，因为不同平台对工具调用的设计不同。例如 Anthropic 文档中区分了 client tools 和 server tools：client tools 由应用侧执行，模型返回 tool_use 后，应用执行工具并返回 tool_result；server tools 则运行在 Anthropic 的基础设施中。

第二步：搜索官方资料，确认 tool calling / function calling 的定义。
第三步：补充 Agent Loop 中工具调用的位置。
第四步：整理工具调用的常见类型。
第五步：加入 AI 评测视角：是否该调、调得准不准、参数对不对、结果用没用对、失败后会不会重试。

风险或不确定性
只靠自己理解写，容易停留在泛泛概念。
比如只写“工具调用让 Agent 更强”，这不够专业。
只看博客，容易被过时说法误导。
这个主题更新很快，最好优先看官方文档。
学习笔记不能只写技术实现。
你是面向 AI 评测岗位准备，所以必须补充“怎么评测工具调用质量”。
不要把工具调用等同于 Agent。
工具调用只是 Agent 能力的一部分。Agent 还包括任务理解、规划、记忆、状态更新、观察结果和循环执行。

## 人工复核

这个案例主要验证 Agent Workflow Prompt 是否能完成完整任务链路。

理想输出应该包含：

1. 正确理解用户目标：写 AI Agent 工具调用学习笔记。
2. 正确判断任务类型：学习笔记规划 + 工具使用判断。
3. 合理判断是否需要搜索：基础概念可不搜索，最新发展、框架案例、行业实践建议搜索。
4. 给出清晰执行计划：先确定笔记结构，再判断资料来源，再补充案例，最后整理成笔记。
5. 能说明工具使用边界：不是所有学习笔记都必须搜索，但涉及最新信息时应搜索。
6. 能说明不确定性：如果要写“当前主流工具调用框架/最新 API 能力”，需要最新资料支撑。

## 复盘

这个案例的重点是练习完整 Agent Prompt，而不是单点工具判断。

任务五的 Agent 工具使用规则 Prompt 解决的是“要不要用工具”。

任务十的 Agent Workflow Prompt 进一步解决：

- 如何理解任务
- 如何拆解步骤
- 如何判断工具
- 如何模拟执行
- 如何更新状态
- 如何判断终止
- 如何输出最终结果

这个案例说明，Agent Prompt 的关键不只是列出工具，还要规定 Agent 的执行流程、错误处理和终止条件。

可以改进的地方：

1. 后续可以加入真实搜索工具调用结果。
2. 可以加入 Agent 执行轨迹日志格式。
3. 可以增加工具调用失败时的示例。
4. 可以增加多轮任务状态更新案例。
