# Skill Case 04：Agent 任务执行

## 1. 使用的 Skill

`skills/agent_task_execution_skill.md`

## 2. 案例目标

判断一个用户任务是否需要工具，并设计 Agent 的执行流程。

## 3. 用户问题

OpenAI API 现在的价格是多少？请帮我整理成表格。

## 4. 适用原因

这个任务涉及“当前价格”，价格可能会变化，不能只依赖模型记忆回答。

该任务需要：

1. 理解用户要查询当前价格；
2. 判断已有信息不足；
3. 判断需要搜索或访问官方页面；
4. 制定查询和整理步骤；
5. 在获得信息后输出表格。

因此适合使用 Agent 任务执行 Skill。

## 5. 任务类型

最新信息查询 + 表格整理。

## 6. 是否需要工具

需要工具。

原因是 OpenAI API 价格属于可能变化的信息，需要查询官方价格页面或可靠来源确认最新数据。

## 7. 预期执行流程

1. 理解用户任务：查询当前 OpenAI API 价格并整理成表格；
2. 判断信息是否充足：不充足，需要实时查询；
3. 判断任务类型：最新信息查询；
4. 判断工具需求：需要搜索工具或网页读取工具；
5. 查询官方价格页面；
6. 提取模型名称、输入价格、输出价格、计费单位；
7. 整理成表格；
8. 标注价格可能变化，以官方页面为准；
9. 输出最终结果。

## 8. 工具使用方案示例

| 步骤 | 工具 | 使用原因 | 输入参数 | 预期结果 |
|---|---|---|---|---|
| 1 | 搜索工具 | 查找最新官方价格页面 | OpenAI API pricing official | 找到官方价格页 |
| 2 | 网页读取工具 | 提取价格信息 | 官方价格页面 | 获得模型价格 |
| 3 | 表格整理 | 方便用户对比 | 模型名、输入价格、输出价格 | 输出价格表 |

## 9. 评测关注点

使用该 Skill 时，需要检查：

- 是否判断该问题需要实时信息；
- 是否避免凭记忆编造价格；
- 是否说明为什么需要工具；
- 是否给出合理执行步骤；
- 是否有当前状态更新；
- 是否有终止判断；
- 是否输出用户真正需要的表格结果。
