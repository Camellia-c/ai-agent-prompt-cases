# Prompt 质量评测 Skill

## 1. Skill 名称

Prompt 质量评测 Skill

## 2. 适用场景

当用户提供一个或多个 Prompt，并希望判断其质量、优劣、适配程度或改进方向时，使用本 Skill。

适用场景包括：

- 对比两个 Prompt 哪个更好；
- 判断某个 Prompt 是否能满足用户需求；
- 找出 Prompt 中的问题；
- 优化一个不够清晰的 Prompt；
- 将普通 Prompt 改成更适合 AI 评测、学习笔记、面试准备或任务执行的 Prompt。

## 3. 用户目标

用户通常不是只想知道“哪个好”，而是想知道：

1. 哪个 Prompt 更适合完成任务；
2. 为什么它更好；
3. 差的 Prompt 问题在哪里；
4. 如何从 Prompt 设计角度改进；
5. 如何形成一套可复用的 Prompt 评测标准。

## 4. 输入格式

用户通常会提供以下内容：

### 类型一：用户需求 + 两个 Prompt

例如：

用户需求：

我想学习 RAG，并希望输出内容适合整理成 AI 评测岗位的学习笔记和面试表达。

Prompt A：

帮我解释一下 RAG。

Prompt B：

你是一个 AI 学习助手。你的任务是面向正在准备 AI 评测岗位的学习者，解释 RAG 是什么。请按照以下结构输出：一句话定义、解决什么问题、基本流程、业务例子、面试问法、1 分钟口头表达版本。

### 类型二：一个 Prompt

例如：

帮我看看这个 Prompt 写得好不好。

### 类型三：需要优化的 Prompt

例如：

把这个 Prompt 改得更适合 AI 评测岗位学习。

## 5. 评测维度

评估 Prompt 时，从以下维度进行判断。

### 维度一：目标清晰度

判断 Prompt 是否明确说明用户真正想完成什么。

重点看：

- 是否说明任务目标；
- 是否说明目标用户；
- 是否说明使用场景；
- 是否避免模糊表达。

### 维度二：角色设定

判断 Prompt 是否给模型设置了合适角色。

重点看：

- 角色是否和任务相关；
- 角色是否能帮助模型更稳定地完成任务；
- 是否存在无意义或过度夸张的角色设定。

### 维度三：任务边界

判断 Prompt 是否明确说明要做什么、不做什么。

重点看：

- 是否说明任务范围；
- 是否限制不相关内容；
- 是否避免模型自由发挥过度；
- 是否要求不确定时说明不确定。

### 维度四：输出结构

判断 Prompt 是否提供清晰的输出格式。

重点看：

- 是否有分点或分层结构；
- 是否要求输出表格、清单、步骤或案例；
- 是否方便用户直接整理、复用或检查。

### 维度五：约束条件

判断 Prompt 是否说明质量要求和限制。

重点看：

- 是否要求不要编造；
- 是否要求语言风格；
- 是否要求面向特定用户水平；
- 是否要求专业词解释；
- 是否要求适配具体岗位或场景。

### 维度六：可执行性

判断模型拿到 Prompt 后是否容易执行。

重点看：

- 指令是否具体；
- 是否存在互相冲突的要求；
- 是否缺少必要信息；
- 是否容易输出空泛内容。

### 维度七：用户需求匹配度

判断 Prompt 是否真正贴合用户需求。

重点看：

- 是否围绕用户原始目标；
- 是否解决用户真实问题；
- 是否考虑用户的学习阶段、使用目的和输出用途。

## 6. 执行流程

### 第一步：理解用户需求

先判断用户真正想完成什么，而不是只看 Prompt 表面文字。

需要明确：

- 用户要解决什么问题；
- 输出要给谁看；
- 输出用于学习、面试、工作、评测还是写作；
- 是否需要结构化结果。

### 第二步：逐个分析 Prompt

分别分析每个 Prompt 的优点和缺点。

分析时不要只说“好”或“不好”，要说明具体原因。

### 第三步：按维度评分

建议使用 1-5 分制。

评分标准：

- 5 分：非常明确、完整、可执行；
- 4 分：整体较好，有少量可优化点；
- 3 分：基本能用，但不够稳定；
- 2 分：问题较多，容易输出泛泛内容；
- 1 分：目标模糊，基本无法稳定完成任务。

### 第四步：给出结论

明确说明哪个 Prompt 更适合用户需求。

如果两个 Prompt 都不够好，需要说明：

- 哪个相对更好；
- 为什么仍然不足；
- 应该如何整合优化。

### 第五步：给出更好的 Prompt 版本

在评估之后，输出一版优化后的 Prompt。

优化版应包含：

- 角色设定；
- 任务目标；
- 输出结构；
- 质量要求；
- 限制条件；
- 用户场景。

## 8. 输出要求

评测 Prompt 时必须遵守以下要求：

1. 不要只给主观评价，要说明依据；
2. 不要只说“更详细所以更好”，要说明详细在哪里；
3. 要围绕用户需求判断，而不是单纯看 Prompt 长短；
4. 如果 Prompt 存在风险，要明确指出；
5. 如果信息不足，要说明需要补充什么；
6. 输出要适合整理成 Prompt 学习笔记；
7. 语言要清楚、直接、可复用。

## 9. 示例输入

用户需求：

我想学习 RAG，并希望输出内容适合整理成 AI 评测岗位的学习笔记和面试表达。

Prompt A：

帮我解释一下 RAG。

Prompt B：

你是一个 AI 学习助手。

你的任务是：面向正在准备 AI 评测岗位的学习者，解释 RAG 是什么。

请按照以下结构输出：

1. 一句话定义
2. RAG 解决了什么问题
3. RAG 的基本流程
4. RAG 和普通大模型问答的区别
5. 一个真实业务例子
6. 面试中可能怎么问
7. 1 分钟口头表达版本

要求：

- 用小白能听懂的话解释；
- 专业词需要解释；
- 不要堆术语；
- 输出适合整理进学习笔记；
- 如果涉及资料中没有的信息，请说明不确定，不要编造。

## 10. 示例输出摘要

### 1. 总体结论

Prompt B 更适合完成用户需求。

原因是 Prompt B 明确了目标用户、任务场景、输出结构和质量要求。它不仅要求解释 RAG，还要求输出内容适合 AI 评测岗位学习和面试表达。相比之下，Prompt A 过于宽泛，只能得到一个普通解释，无法保证输出结构和岗位适配度。

### 2. 分项评分表

| 维度 | Prompt A | Prompt B | 说明 |
|---|---:|---:|---|
| 目标清晰度 | 2 | 5 | A 只说解释 RAG，B 明确面向 AI 评测岗位学习者 |
| 角色设定 | 1 | 4 | A 没有角色，B 设置了 AI 学习助手 |
| 输出结构 | 1 | 5 | A 没有结构，B 明确了 7 个输出模块 |
| 约束条件 | 1 | 5 | A 没有要求，B 要求小白可懂、解释术语、不编造 |
| 可执行性 | 3 | 5 | A 可以执行但结果不稳定，B 执行路径清晰 |
| 用户需求匹配度 | 2 | 5 | B 明显更贴合学习笔记和面试表达需求 |

### 3. Prompt A 分析

Prompt A 的优点是简单直接，模型可以快速回答。但问题也很明显：它没有说明用户的学习目标、输出结构、岗位背景和质量要求，所以模型大概率只会输出一段通用解释。

### 4. Prompt B 分析

Prompt B 的优势在于任务边界清楚。它明确了用户是 AI 评测岗位学习者，也规定了输出结构和语言要求，因此更容易得到可整理、可复习、可面试复述的结果。

### 5. 主要差异

两个 Prompt 的核心差异不是长短，而是任务设计完整度。Prompt B 把“解释 RAG”升级成了“面向特定学习目标的结构化输出”。

### 6. 修改建议

Prompt A 可以补充角色、用户背景、输出结构和质量要求。例如明确要求“面向 AI 评测岗位学习者”“输出适合学习笔记”“包含面试表达版本”。

### 7. 更好的 Prompt 版本

你是一个 AI 学习资料整理助手。

你的任务是：面向正在准备 AI 评测岗位的学习者，解释 RAG 是什么，并整理成适合学习笔记和面试复述的内容。

请按照以下结构输出：

1. 一句话定义；
2. RAG 解决的问题；
3. RAG 的基本流程；
4. RAG 和普通大模型问答的区别；
5. RAG 在 AI 评测中的关注点；
6. 常见错误和风险；
7. 面试可能怎么问；
8. 1 分钟口头表达版本。

要求：

- 用初学者能理解的语言；
- 专业词要解释；
- 不要堆术语；
- 不要编造不确定信息；
- 输出要适合直接整理进学习笔记。
