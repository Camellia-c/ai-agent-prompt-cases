# Agent 任务执行 Skill

## 1. Skill 名称

Agent 任务执行 Skill

## 2. 适用场景

当用户提出一个需要拆解、规划、判断工具使用或多步骤执行的任务时，使用本 Skill。

适用场景包括：

- 判断一个任务是否需要调用工具；
- 拆解复杂任务的执行步骤；
- 模拟 AI Agent 的任务执行流程；
- 训练 Agent Loop 思维；
- 将用户问题整理成可执行方案；
- 面向 AI 评测岗位分析 Agent 的执行质量。

## 3. 用户目标

用户通常不是只想得到一个简单答案，而是希望知道：

1. 这个任务到底要完成什么；
2. 当前信息是否充足；
3. 是否需要调用外部工具；
4. 应该按照什么步骤执行；
5. 每一步需要什么输入和输出；
6. 执行过程中如何观察结果；
7. 什么时候继续，什么时候终止。

## 4. 核心思想

AI Agent 不是只生成一段回答，而是围绕任务进行持续执行。

一个典型 Agent 执行流程包括：

1. 理解任务；
2. 判断信息是否充足；
3. 制定计划；
4. 判断是否需要工具；
5. 调用工具或模拟执行；
6. 观察执行结果；
7. 更新当前状态；
8. 判断是否继续；
9. 输出最终结果。

这套流程也可以理解为 Agent Loop。

## 5. 输入类型

### 类型一：概念类问题

例如：

AI Agent 为什么需要工具调用？

这类问题通常不需要工具，主要靠解释和结构化表达完成。

### 类型二：最新信息查询

例如：

OpenAI API 现在的价格是多少？

这类问题需要搜索工具，因为价格可能变化，不能只靠模型记忆回答。

### 类型三：文件分析任务

例如：

帮我分析这个 Excel 表格里的销售数据。

这类任务需要文件读取工具、表格工具或代码执行工具。

### 类型四：数据计算任务

例如：

帮我计算这批学生的续费风险分，并按高、中、低风险分类。

这类任务需要计算工具或代码执行工具。

### 类型五：多步骤综合任务

例如：

帮我设计一个 AI 评测作品集项目，并生成 README、案例和面试表达。

这类任务需要拆解计划，分阶段执行。

## 6. 工具判断规则

### 不需要工具的情况

以下情况通常不需要工具：

- 解释稳定概念；
- 总结用户已提供的文本；
- 改写文案；
- 生成学习笔记；
- 制定通用计划；
- 分析不依赖最新信息的问题。

### 需要工具的情况

以下情况通常需要工具：

- 查询最新信息；
- 查询价格、政策、版本、新闻；
- 阅读用户上传的文件；
- 处理 Excel、CSV、PDF 等文件；
- 进行精确计算；
- 运行代码；
- 生成图表；
- 操作邮件、日历、数据库或外部系统。

### 信息不足时的处理

如果任务信息不足，需要明确说明缺少什么。

例如：

- 缺少文件；
- 缺少字段含义；
- 缺少时间范围；
- 缺少评价标准；
- 缺少目标对象；
- 缺少输出格式要求。

如果缺失信息不影响初步规划，可以先给出可执行方案，并标注假设条件。

## 7. 执行流程

### 第一步：任务理解

先用自己的话说明用户真正想完成什么。

### 第二步：信息充足性判断

判断完成任务需要的信息是否已经具备。

输出：

- 充足；
- 基本充足；
- 不充足。

如果不充足，要说明缺少哪些信息。

### 第三步：任务类型判断

将任务归类，例如：

- 概念解释；
- 最新信息查询；
- 文件分析；
- 数据计算；
- 代码执行；
- API 查询；
- 邮件日历操作；
- 多步骤综合任务。

### 第四步：制定执行计划

将任务拆成清晰步骤。

要求：

- 每一步有明确目的；
- 顺序合理；
- 能执行；
- 不要泛泛而谈。

### 第五步：判断是否需要工具

明确说明：

- 需要工具；
- 不需要工具。

如果需要，要说明需要什么工具，以及为什么。

### 第六步：工具使用方案

如果需要工具，输出工具使用表格。

| 步骤 | 工具 | 使用原因 | 输入参数 | 预期结果 |
|---|---|---|---|---|

如果不需要工具，写：

无需使用工具。

### 第七步：执行过程或模拟执行

如果可以直接执行，就根据现有信息给出结果。

如果不能真正执行，就说明模拟执行过程。

要求：

- 不假装已经调用了没有调用的工具；
- 不编造工具结果；
- 不把假设当事实。

### 第八步：当前状态更新

说明已经完成什么，还剩什么。

### 第九步：终止判断

判断任务是否可以结束。

输出：

- 可以结束：说明原因；
- 需要继续：说明下一步要做什么。

### 第十步：最终输出

给用户一个清晰、可执行、可复用的结果。

使用本 Skill 时，按照以下格式输出：

充足 / 基本充足 / 不充足。

如果需要工具，使用表格：

| 步骤 | 工具 | 使用原因 | 输入参数 | 预期结果 |
|---|---|---|---|---|

如果不需要工具，写：

无需使用工具。

## 7. 执行过程或模拟执行

说明每一步如何执行。

## 8. 当前状态更新

说明已完成什么、还缺什么。

## 9. 终止判断

说明任务是否可以结束，或者是否需要继续执行。

## 10. 最终结果

给出用户可以直接使用的结果。

## 9. 输出要求

使用本 Skill 时必须遵守以下要求：

1. 不要跳过任务理解；
2. 不要在信息不足时强行编造；
3. 不要假装调用了没有调用的工具；
4. 工具判断要有理由；
5. 执行计划要具体；
6. 状态更新要清楚；
7. 终止判断要明确；
8. 输出要适合整理成 AI Agent 学习笔记；
9. 语言要清楚、直接、可执行。

## 10. 示例输入

用户问题：

OpenAI API 现在的价格是多少？

## 11. 示例输出摘要

### 1. 任务理解

用户想了解 OpenAI API 当前价格。这个问题涉及最新价格信息，价格可能随时间变化，不能只依赖模型记忆回答。

### 2. 信息充足性判断

不充足。

缺少实时价格数据，需要查询 OpenAI 官方价格页面或可靠来源。

### 3. 任务类型

最新信息查询。

### 4. 执行计划

1. 确认用户想查询的是哪类模型价格；
2. 查询 OpenAI 官方价格信息；
3. 提取输入 token、输出 token 或其他计费维度；
4. 整理成表格；
5. 提醒用户价格可能变化，以官方页面为准。

### 5. 是否需要工具

需要。

原因是 API 价格属于可能变化的信息，需要使用搜索工具或访问官方文档确认最新数据。

### 6. 工具使用方案

| 步骤 | 工具 | 使用原因 | 输入参数 | 预期结果 |
|---|---|---|---|---|
| 1 | 搜索工具 | 获取最新官方价格 | OpenAI API pricing official | 找到官方价格页面 |
| 2 | 网页读取工具 | 提取模型价格 | 官方价格页面 | 获得各模型计费信息 |
| 3 | 表格整理 | 方便用户对比 | 模型名称、输入价格、输出价格 | 输出价格对比表 |

### 7. 执行过程或模拟执行

当前没有真实查询结果时，不能直接编造价格。需要先访问官方页面，再基于查询结果整理。

### 8. 当前状态更新

已完成任务判断和执行计划。尚未获得最新价格数据。

### 9. 终止判断

不能直接结束。下一步需要查询官方价格页面。

### 10. 最终结果

这个任务需要联网查询。查询后应输出一份包含模型名称、输入价格、输出价格、计费单位和注意事项的价格表。

## 12. Skill 质量检查清单

使用本 Skill 输出内容后，需要检查：

- 是否准确理解用户任务；
- 是否判断信息是否充足；
- 是否正确判断任务类型；
- 是否合理判断工具需求；
- 是否没有编造工具结果；
- 是否给出清晰执行计划；
- 是否有状态更新；
- 是否有终止判断；
- 是否适合整理成 AI Agent 学习笔记。
