# AI 学习资料整理 Skill

## 1. Skill 名称

AI 学习资料整理 Skill

## 2. 适用场景

当用户提供一段 AI / 大模型 / AI Agent / AI 评测相关资料时，本 Skill 用于将资料整理成适合学习、复习和面试表达的结构化笔记。

适用内容包括但不限于：

- LLM 基础概念
- RAG
- AI Agent
- Tool Use / Function Calling
- Agent Loop
- Memory
- Guardrails
- Evaluation
- Debugging
- Prompt Engineering
- SFT / RLHF / DPO
- Embedding / Token / Chunk
- Transformer
- 多模态
- 模型部署与优化

## 3. 用户目标

用户不是单纯想“看懂概念”，而是希望：

1. 理解这个概念是什么；
2. 知道它解决什么问题；
3. 知道它在真实业务中怎么用；
4. 知道它和 AI 评测岗位有什么关系；
5. 能整理成学习笔记；
6. 能在面试中讲出来。

## 4. 输入类型

用户通常会提供以下三类输入。

### 类型一：原始资料

例如：

RAG 指的是 Retrieval-Augmented Generation，也就是检索增强生成。它通过外部知识库检索相关内容，再交给大模型生成回答。

### 类型二：一个概念名称

例如：

帮我整理一下 Tool Use。

### 类型三：一段不完整理解

例如：

我理解 Agent Loop 就是模型一直循环执行任务，对吗？

## 5. 执行流程

### 第一步：判断资料是否充足

先判断用户提供的信息是否足够完成整理。

如果信息充足，直接整理。

如果信息不足，需要明确说明：

- 哪些部分可以基于现有资料整理；
- 哪些部分资料不足；
- 哪些内容不能自行编造；
- 是否需要额外补充资料。

### 第二步：提炼核心结论

用 1-3 句话说明这个概念的本质。

要求：

- 不堆术语；
- 不绕弯子；
- 优先解释“它到底是干什么的”；
- 适合初学者理解。

### 第三步：解释关键概念

将资料中的专业词拆开解释。

要求：

- 适合初学者理解；
- 不直接照抄原文；
- 必要时使用类比；
- 解释后要能用于面试表达。

### 第四步：梳理流程或机制

如果该概念涉及流程，需要整理为步骤。

例如 RAG 的流程可以整理为：

1. 用户提出问题；
2. 系统去知识库中检索相关资料；
3. 将检索结果和用户问题一起交给大模型；
4. 大模型基于资料生成回答；
5. 系统返回答案。

### 第五步：关联 AI 评测岗位

说明该概念在 AI 评测岗位中为什么重要。

可以从以下角度分析：

- 评测什么；
- 容易出现什么问题；
- 如何判断输出质量；
- 需要关注哪些指标；
- 面试中可能怎么问；
- 真实业务中如何发现问题。

### 第六步：输出面试表达版本

将复杂概念压缩成 1 分钟左右可以口头表达的内容。

要求：

- 像真人面试回答；
- 不像背百科；
- 有逻辑顺序；
- 能体现岗位理解；
- 不要堆砌概念。

## 6. 标准输出结构

每次使用本 Skill 时，按照以下结构输出：

## 1. 核心结论

用 1-3 句话说明这个概念的本质。

## 2. 关键概念解释

解释资料中的重要术语或关键点。

## 3. 重要流程或机制

如果涉及流程，按步骤说明。

## 4. 和 AI 评测岗位的关系

说明它为什么和 AI 评测有关。

## 5. 常见问题或风险点

说明这个概念在真实业务中容易出现的问题。

## 6. 面试可能怎么问

列出 3-5 个可能的面试问题。

## 7. 1 分钟口头表达版本

给出一段适合面试时直接说出来的回答。

## 7. 输出要求

整理内容时必须遵守以下要求：

1. 不要照抄原文；
2. 不要编造资料中没有的信息；
3. 专业词必须解释；
4. 不要堆概念，要说明“有什么用”；
5. 输出要适合放进学习笔记；
6. 输出要适合面试复述；
7. 语言要清楚、直接、口语化；
8. 如果资料不足，要明确说“不确定”或“资料中没有体现”。

## 8. 示例输入

RAG 指的是 Retrieval-Augmented Generation，也就是检索增强生成。它通过外部知识库检索相关内容，再交给大模型生成回答。RAG 可以缓解模型知识过时、幻觉和无法访问私有数据的问题。

## 9. 示例输出

### 1. 核心结论

RAG 是一种让大模型先查资料、再回答问题的方法。它不是单纯依赖模型记忆，而是把外部知识库中的内容检索出来，再交给模型生成答案。

### 2. 关键概念解释

RAG 的英文是 Retrieval-Augmented Generation，意思是“检索增强生成”。

其中：

- Retrieval 是检索，也就是先从知识库、文档库或数据库中找相关资料；
- Augmented 是增强，表示用检索到的资料增强模型回答能力；
- Generation 是生成，也就是大模型根据问题和资料生成最终回答。

### 3. 重要流程或机制

RAG 的典型流程是：

1. 用户提出问题；
2. 系统将问题转成适合检索的形式；
3. 从知识库中找到相关文档片段；
4. 把这些文档片段和用户问题一起放进 Prompt；
5. 大模型根据这些内容生成回答；
6. 系统返回最终答案。

### 4. 和 AI 评测岗位的关系

RAG 和 AI 评测关系很密切，因为 RAG 系统不仅要评测模型回答是否通顺，还要评测它有没有正确使用检索资料。

AI 评测时通常需要关注：

- 检索内容是否相关；
- 模型有没有引用错误资料；
- 回答是否基于资料；
- 是否出现幻觉；
- 是否遗漏关键事实；
- 是否把资料理解错。

### 5. 常见问题或风险点

RAG 常见问题包括：

1. 检索不到正确资料；
2. 检索到了资料，但模型没有正确使用；
3. 检索资料本身过时或错误；
4. 模型编造资料中没有的内容；
5. 回答看起来很流畅，但事实不准确。

### 6. 面试可能怎么问

1. RAG 是什么？
2. RAG 解决了大模型的什么问题？
3. RAG 和普通大模型问答有什么区别？
4. 你会如何评测一个 RAG 系统？
5. RAG 系统常见的错误有哪些？

### 7. 1 分钟口头表达版本

我理解 RAG 是一种让大模型“先查资料再回答”的技术。普通大模型主要依赖训练时学到的知识，所以可能会遇到知识过时、无法访问企业私有数据、容易幻觉等问题。RAG 会先从外部知识库里检索和用户问题相关的内容，然后把这些内容和问题一起交给模型，让模型基于资料生成回答。

如果从 AI 评测角度看，RAG 不能只看回答是否通顺，还要看检索结果是否相关、模型有没有忠实使用资料、有没有编造资料中不存在的信息，以及最终答案是否真正解决了用户问题。

## 10. Skill 质量检查清单

使用本 Skill 输出内容后，需要检查：

- 是否解释清楚“是什么”；
- 是否说明“解决什么问题”；
- 是否讲清楚“怎么工作”；
- 是否联系到 AI 评测岗位；
- 是否有面试表达；
- 是否避免了编造；
- 是否适合用户整理进学习笔记。
